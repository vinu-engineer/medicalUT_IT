Detailed Description

Implementation of alert record generation from vital sign readings.

Implements generate_alerts() declared in alerts.h. Each abnormal parameter produces exactly one Alert record written into the caller's buffer. The internal PUSH_ALERT macro encapsulates the bounds check and string formatting to keep the function body concise and auditable.

Implementation Notes

  • _CRT_SECURE_NO_WARNINGS suppresses MSVC deprecation warnings for strncpy and snprintf. These functions are used with explicit length limits and guaranteed null-termination, so the usage is safe.
  • The macro parameter is named _al (not level) to avoid shadowing the struct field Alert.level inside the macro expansion body.

Generate structured alert records for out-of-range vital signs.

Generate alert records for every vital sign parameter outside its normal clinical range.

Internal flow for each parameter:

  1. Call the corresponding check_*() function.
  2. If the result is not ALERT_NORMAL, write an Alert record via PUSH_ALERT.
  3. PUSH_ALERT enforces the max_out cap before writing.

The PUSH_ALERT macro copies the parameter name with strncpy (max 31 chars

  • null) and formats the message with snprintf (max ALERT_MSG_LEN - 1 chars
  • null), guaranteeing null-termination in both fields.
