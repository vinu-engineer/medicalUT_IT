Detailed Description

Implementation of patient record management functions.

Implements all functions declared in patient.h. The module uses only static memory (no heap allocation) in compliance with IEC 62304 Class B guidance for embedded medical device software.

Memory Safety

  • patient_init() zero-fills the entire record before writing fields, preventing stale data from a previous session.
  • strncpy is used with an explicit length limit and explicit null-termination to guard against buffer overrun.
  • patient_add_reading() enforces the MAX_READINGS cap and returns a status code so callers can detect and handle overflow.
