Detailed Description

Vital signs data types, thresholds, and validation functions.

This module defines the core data structures and validation logic for patient vital signs. Each parameter is evaluated against clinically defined thresholds derived from AHA/ACC guidelines and classified into one of three alert levels: NORMAL, WARNING, or CRITICAL.

Threshold Reference Table

Parameter Normal Warning Critical
Heart Rate 60–100 bpm 41–59 / 101–150 bpm ≤40 / ≥151 bpm
Systolic BP 90–140 mmHg 71–89 / 141–180 mmHg ≤70 / ≥181 mmHg
Diastolic BP 60–90 mmHg 41–59 / 91–120 mmHg ≤40 / ≥121 mmHg
Temperature 36.1–37.2 °C 35.0–36.0 / 37.3–39.5 °C <35.0 / >39.5 °C
SpO2 95–100 % 90–94 % <90 %

◆ AlertLevel

enum AlertLevel

Classification of a vital sign reading relative to clinical norms.

The numeric ordering is intentional: ALERT_NORMAL < ALERT_WARNING < ALERT_CRITICAL, allowing comparisons such as if (level > ALERT_NORMAL). This ordering is relied upon by overall_alert_level().

Enumerator
ALERT_NORMAL 

Reading within normal clinical range.

ALERT_WARNING 

Reading outside normal range; clinician review recommended.

ALERT_CRITICAL 

Reading at potentially life-threatening level; immediate intervention required.

◆ check_blood_pressure()

AlertLevel check_blood_pressure ( int systolic,
int diastolic )

Classify a blood pressure reading against clinical thresholds.

Both systolic and diastolic values are evaluated independently. The more severe of the two classifications is returned.

  • NORMAL: Systolic 90–140 mmHg AND diastolic 60–90 mmHg
  • WARNING: Systolic 71–89 / 141–180 mmHg OR diastolic 41–59 / 91–120 mmHg
  • CRITICAL: Systolic ≤70 / ≥181 mmHg OR diastolic ≤40 / ≥121 mmHg
Parameters
[in]systolicSystolic pressure in mmHg.
[in]diastolicDiastolic pressure in mmHg.
Returns
AlertLevel classification for the combined blood pressure reading.
Requirement
SWR-VIT-002

Classify a blood pressure reading against clinical thresholds.

Implements SWR-VIT-002 threshold logic. Critical thresholds are evaluated first (fail-safe ordering). If either systolic or diastolic is critical, CRITICAL is returned. If either is outside the normal range, WARNING is returned.

◆ overall_alert_level()

AlertLevel overall_alert_level ( const VitalSigns * vitals)

Return the highest alert level across all vital sign parameters.

Internally calls check_heart_rate(), check_blood_pressure(), check_temperature(), and check_spo2(), then returns the maximum value. This ensures that any single critical parameter elevates the overall patient status to CRITICAL regardless of other parameters.

Parameters
[in]vitalsPointer to a valid VitalSigns structure. Must not be NULL.
Returns
The highest AlertLevel among all four parameters.
Precondition
vitals != NULL
Requirement
SWR-VIT-005

Return the highest alert level across all vital sign parameters.

Implements SWR-VIT-005. Iterates all four checks and tracks the maximum. Uses a fixed-size array to avoid branch explosion and keep cyclomatic complexity low for IEC 62304 review.
