5 * @details
6 * Provides hw_init() and hw_get_next_reading() using a fixed 20-entry
7 * clinical scenario table that cycles through four phases:
8 *
9 * Index 0– 4 : STABLE NORMAL — resting, all parameters within range
10 * Index 5– 8 : DETERIORATING — gradual rise into WARNING territory
11 * Index 9–11 : CRITICAL — life-threatening values
12 * Index 12–18 : RECOVERING — progressive return to normal
13 * Index 19 : STABLE NORMAL — cycle repeats
14 *
15 * To connect real hardware, replace this file with hw_driver.c that
16 * implements the same two function signatures without modifying any
17 * other source file (see hw_vitals.h for the full contract).
18 *
19 * All storage is static — no heap allocation (SYS-012).
