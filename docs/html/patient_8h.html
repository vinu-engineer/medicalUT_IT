Detailed Description

Patient record management — demographic data and vital sign history.

This module provides the top-level data structure for a monitored patient and the functions to manage their vital sign history. It composes the vitals and alerts modules to provide a complete patient-status API.

Architecture Position

PatientRecord sits at the top of the module hierarchy:

patient.h
└── vitals.h (classification of individual readings)
└── alerts.h (alert record generation)

Storage Model

Readings are stored in a fixed-size array (MAX_READINGS = 10). Once full, further additions are rejected (patient_add_reading() returns 0). This design avoids dynamic memory allocation, a requirement for safety-critical embedded software under IEC 62304.

int patient_add_reading ( PatientRecord * rec,
const VitalSigns * v )

Append a vital sign reading to the patient's history.

The VitalSigns structure is copied by value into the internal array. If the record is already full (reading_count == MAX_READINGS), the record is left unchanged and 0 is returned.

Parameters
[in,out]recPointer to an initialised PatientRecord. Must not be NULL.
[in]vPointer to the VitalSigns reading to append. Copied by value; the caller may free or reuse v afterwards.
Returns
1 if the reading was successfully appended, 0 if the buffer is full.
Precondition
rec != NULL
v != NULL
Postcondition
On success: rec->reading_count is incremented by 1.
On failure: rec is unchanged.
Requirement
SWR-PAT-002

Append a vital sign reading to the patient's history.

VitalSigns is copied by value (struct assignment), so the caller retains ownership of the original and may reuse the variable.

Initialise a PatientRecord with demographic data.

Zero-fills the entire structure before populating fields. reading_count is set to 0. The name string is copied with truncation to MAX_NAME_LEN - 1 characters and guaranteed null-termination.

Parameters
[out]recPointer to the record to initialise. Must not be NULL.
[in]idUnique patient identifier.
[in]nameNull-terminated patient name string. Must not be NULL.
[in]agePatient age in years.
[in]weight_kgBody weight in kilograms.
[in]height_mHeight in metres.
Precondition
rec != NULL
name != NULL
Postcondition
rec->reading_count == 0
Requirement
SWR-PAT-001

Initialise a PatientRecord with demographic data.

memset(0) ensures that any unused readings array slots contain all-zero VitalSigns, which classify as CRITICAL across all parameters — a deliberately safe default that prevents silent display of uninitialised data.
