Detailed Description

Alert record generation from a set of vital sign measurements.

This module is responsible for translating a VitalSigns observation into a structured list of Alert records — one per abnormal parameter. It acts as the bridge between raw clinical data (vitals.h) and the patient management layer (patient.h).

Design Decisions

  • Each abnormal parameter produces exactly one Alert record.
  • The output buffer size is controlled by the caller via max_out, making the function safe for use in memory-constrained embedded targets.
  • All-normal vitals produce zero alerts (return value 0).
  • Alert messages are human-readable and suitable for display on a clinical workstation or serial console.

IEC 62304 Traceability

  • Software Unit: UNIT-ALT
  • Requirements covered: SWR-ALT-001 through SWR-ALT-004

int generate_alerts ( const VitalSigns * vitals,
Alert * out,
int max_out )

Generate alert records for every vital sign parameter outside its normal clinical range.

Iterates over all four vital sign parameters in this fixed order:

  1. Heart rate
  2. Blood pressure (systolic / diastolic)
  3. Temperature
  4. SpO2

For each parameter that is not ALERT_NORMAL, one Alert entry is written to out. Writing stops when either all parameters have been checked or max_out entries have been filled — whichever comes first.

Memory safety: the function never writes beyond out[max_out - 1]. Passing max_out = 0 is valid and writes nothing.

Parameters
[in]vitalsPointer to a fully populated VitalSigns structure. Must not be NULL.
[out]outCaller-allocated array to receive Alert records. Must hold at least max_out elements.
[in]max_outMaximum number of Alert entries to write.
Returns
Number of Alert records written to out (0 if all parameters are normal, or if max_out == 0).
Precondition
vitals != NULL
out != NULL || max_out == 0
Postcondition
Return value <= max_out
Requirements
SWR-ALT-001, SWR-ALT-002, SWR-ALT-003, SWR-ALT-004

Generate alert records for every vital sign parameter outside its normal clinical range.

Internal flow for each parameter:

  1. Call the corresponding check_*() function.
  2. If the result is not ALERT_NORMAL, write an Alert record via PUSH_ALERT.
  3. PUSH_ALERT enforces the max_out cap before writing.

The PUSH_ALERT macro copies the parameter name with strncpy (max 31 chars

  • null) and formats the message with snprintf (max ALERT_MSG_LEN - 1 chars
  • null), guaranteeing null-termination in both fields.
